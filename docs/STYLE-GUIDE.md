# SK AutoSphere Style Guide

This comprehensive style guide documents the design system for SK AutoSphere. It serves as the source of truth for designers and developers to ensure consistency across the platform.

## 1. Brand Identity & Color Palette

The brand is anchored by a vibrant effortless blue, communicating trust and modern technology.

### Primary Brand Color

The core color is **Electric Blue**.

- **Hex:** `#2558fa`
- **RGB:** `37, 88, 250`
- **Usage:** Primary buttons, active states, key brand moments, headers.

### Color Scale (Best Practices)

Generated from the primary brand color to ensure accessibility and depth.

| Token             | Hex       | Usage                                           |
| :---------------- | :-------- | :---------------------------------------------- |
| **Primary**       | `#2558fa` | Main Brand Color.                               |
| **Primary Dark**  | `#1a3ec1` | Hover states, active pressed states.            |
| **Primary Light** | `#dbeafe` | Subtle backgrounds, active selection highlight. |
| **Surface**       | `#ffffff` | Card backgrounds, Elevated areas.               |
| **Background**    | `#f8fafc` | Main app background (Slate-50).                 |

_Note: In code, we map these to Tailwind `primary` utility classes._

## 2. Brand Patterns & Textures

### The "Plus Grid" Pattern

A signature background pattern used to add texture without noise. It represents precision and connectivity.

- **Visual:** A repeating grid of small, low-opacity plus (`+`) signs.
- **Usage:** Hero section backgrounds, accented container backgrounds.
- **Implementation:**

### Gradient Usage

- **Brand Gradient:** `bg-gradient-to-br from-[#2558fa] via-[#4f7aff] to-[#7c9dff]`
- Used in conjunction with the Plus Grid pattern for high-impact areas (Hero banners, CTA cards).

## 3. Typography System

**Font Family:** `Geist Sans` (Modern, geometric, legible).

### Type Scale

- **Display:** `text-4xl` to `text-6xl`, font-bold, tracking-tight.
- **Headings:** `text-xl` to `text-3xl`, font-semibold.
- **Body:** `text-base` (16px), relaxed line-height.
- **Small:** `text-sm` (14px), used for metadata and secondary text.

## 4. Component Library

### Buttons

**Path:** `components/ui/button.tsx`

| Variant       | Visuals                                                     |
| :------------ | :---------------------------------------------------------- |
| **Default**   | `bg-[#2558fa] text-white hover:bg-[#1a3ec1]`                |
| **Secondary** | `bg-slate-100 text-slate-900 hover:bg-slate-200`            |
| **Outline**   | `border-2 border-[#2558fa] text-[#2558fa] hover:bg-blue-50` |
| **Ghost**     | `text-slate-600 hover:bg-slate-100`                         |

### Cards

**Path:** `components/ui/card.tsx`

- **Style:** Clean, white surface with subtle shadow.
- **Shadow:** `shadow-sm` for lists, `shadow-lg` for floating elements.
- **Border:** `border-slate-200` (Light grey).

### Badges

**Path:** `components/ui/badge.tsx`

- Used for status (e.g., "Verified", "Sold").
- **Style:** Rounded-full or rounded-md, usually using `bg-primary/10 text-primary` for a lighter feel.

## 5. Visual Hierarchy & Spacing

### Spacing

- **Base Unit:** 4px.
- **Section Padding:** `py-12` or `py-16` (48px - 64px).
- **Component Gap:** `gap-4` or `gap-6`.

### Radius

- **Cards/Containers:** `rounded-xl` (Modern, friendly).
- **Buttons/Inputs:** `rounded-md` (Functional, crisp).
